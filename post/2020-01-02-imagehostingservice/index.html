Use github for image hosting service

Posted by     Jing on Thursday, January 2, 2020

When we write something with Markdown, we frequently need to inset images to Markdown files with ![xxx](url) or ![xxx](relative path of picture). I recommand to use ![xxx](url) to add images due to changes of image paths when we migrate Blogs. However, if we use the first one, it would be better because image url does not ofen change. Github can provide us to use repositories to store our images for free. So I introduce two methods to use github for image hosting service.

Method 1

  • create a new repository

  • upload local images

  • click a image

  • check the image link which would be https://github.com/username/xxxrepository/blob/master/xxx.png

  • replace blob with raw

  • copy the new link to your markdown file

Example ![](https://github.com/shanyit/ImageHosting/raw/master/example/lin.jpeg)

Method 2

  • create a new repository

  • open a new issue

  • drag your local picture to the issue

  • copy the produced link to your markdown file

Example ![lin2](https://user-images.githubusercontent.com/59395853/71666096-72a74600-2d9a-11ea-8520-7ce0139296bb.jpg)
